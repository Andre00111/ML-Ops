Job-by-Job Breakdown

1️⃣ Job 1: Detect Changes

Zweck: Prüfe ob Training notwendig ist

Was passiert:

  • Vergleiche Dateien mit dem letzten Commit
  • Suche nach Änderungen in: src/train.py, src/evaluate.py, data/
  • Wenn Änderungen gefunden → should_train=true
  • Wenn keine Änderungen → should_train=false (Training wird übersprungen)

Output: Flag für nächste Jobs

Duration: ~30 sec

2️⃣ Job 2: Train

Zweck: Trainiere das ML-Modell

Abhängigkeiten: Wartet auf Job 1 (Detect Changes)

Was passiert:

1. Checkout Code von GitHub
2. Setup Python 3.11 Environment
3. pip install -r requirements.txt
   → scikit-learn, pandas, joblib, pytest, flask
4. python src/train.py
   → Lädt Iris Dataset (150 Samples)
   → Teilt auf: 120 Training, 30 Test
   → Trainiert Logistic Regression Modell
   → Speichert zu: models/model.pkl
5. GitHub Actions uploaded Artifact (30 Tage Speicher)

Output:

Model trained and saved to models/model.pkl
Training set size: 120
Test set size: 30
✓ Success

Artifact: model.pkl (991 bytes)

Duration: ~1 min

3️⃣ Job 3: Evaluate

Zweck: Quality Gate — ist das Modell gut genug?

Abhängigkeiten: Wartet auf Job 2 (Train)

Was passiert:

1. Download model.pkl vom Artifact Storage
2. python src/evaluate.py
   → Lädt Iris Dataset mit gleicher 80/20 Aufteilung
   → Lädt trainiertes Modell
   → Evaluiert auf Test-Set (30 Samples)
   → Berechnet Metriken:
      - Accuracy: 1.0000 (100%)
      - Precision pro Klasse: 1.0
      - Recall pro Klasse: 1.0
      - F1 pro Klasse: 1.0
      - Confusion Matrix: Perfekt (keine Fehler)
3. QUALITY GATE CHECK:
   → Wenn Accuracy < 0.70 → JOB FAILS ❌
   → Wenn Accuracy ≥ 0.70 → JOB PASSES ✓
4. Speichert Metriken zu: results/metrics.json
5. Uploaded Artifact

Output:

{
  "accuracy": 1.0,
  "precision": [1.0, 1.0, 1.0],
  "recall": [1.0, 1.0, 1.0],

Artifact: metrics.json

4️⃣ Job 4: Inference Test

Zweck: Teste dass Modell Predictions machen kann

Abhängigkeiten: Wartet auf Job 3 (Evaluate)

Was passiert:

⚡ Was in echten Production Pipelines ANDERS ist

Deine Pipeline hat die richtige Architektur. In echten Production Umgebungen wäre es hauptsächlich komplexer wegen:

1. Daten-Magnitude

Dein Projekt: 150 Samples (Iris Dataset)

Production: Millionen bis Milliarden Samples

Konsequenz: Data Fetching allein dauert 30-60 Minuten statt 0 Minuten

2. Data Validation

Dein Projekt: Keine Validierung (Schema ist garantiert ok)

Production: 5+ Validierungsebenen (Schema, Stats, Quality, Leakage Detection, Drift)

Konsequenz: +30-60 Minuten Pipeline Time

3. Training Komplexität

Dein Projekt: Logistic Regression, 1 Sekunde Training

Production: XGBoost/Deep Learning mit Hyperparameter Tuning (100+ Kombinationen), GPU, 4-8 Stunden Training

Konsequenz: 240-480 Minuten länger (~360x)

4. Advanced Testing

Dein Projekt: 5 Unit Tests

Production: 9+ Tests (Fairness, Bias, Adversarial, Robustness, Latency, Regression, etc.)

Konsequenz: +30-45 Minuten Pipeline Time

5. Monitoring & Safety

Dein Projekt: Keine Monitoring

Production: 24/7 Monitoring, Automatic Alerts, Auto-Rollback wenn Modell degradiert

Konsequenz: Continuous überwachen nach Deployment

6. Deployment Strategy

Dein Projekt: Deployment nicht implementiert

Production: Canary Deployment (5% Traffic → 100% Traffic über 24 Stunden), mit Rollback Plan

Konsequenz: +1440 Minuten für sichere Deployment
